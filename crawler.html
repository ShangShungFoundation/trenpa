crawler.check_backup.do_check_backup(load_files_to_be_checked, storage_id, db_directory_path, backup_directory_path, test_checksum=None)¶

Checks, whether for all the metadata stored in database exists the files in a directory

load_files_to_be_checked - function that given a path will retrieve the list of files to be checked storage_id - check files retrieved from this storage db_directory_path - all the files, that are directly or indirectly in this directory, that means has this prefix in its path will be checked for existence backup_directory_path - directory, where the backed up files are stored. test_checksum - optional hint, whether the program should check the checksum for the files

crawler.tika_parser.translate_tika_to_hachoir(metadata)¶

Since the keys for same values have different names in both tika and hachoir, we want to convert them to one name.

For example:
tika has field: “Content-Type”: “application/pdf” hachoir has field: “mime_type”: “image/jpeg” => both the values will be under the “mime_type”
